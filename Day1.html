Review Homework

How does a computer work?

    1. Input - Keyboard, Mouse, Sensors, Cameras, Touch Screens, etc.
    2. Store Infomration - Memory (RAM) or Hard Drive
    3. Process Information - CPU, Graphics Card
    4. Output Results, Monitor, Speakers, Robots, etc.

In a program we usually:

  1. Ask for input
  2. Process and or store the information
  3. Output results

We are mimicing the hardware at a higher level. This makes programming and abstract exercise. Instead of directly manipulating the hardware, we are manipulating our program that is then translated into a format that the hardward/machine understands. There are often multiple layers of abstraction between the code we write and the steps the computer performs.

The key is to recognize what the proper level of abstraction is so that we can issue efficient instructions. Too much detail wastes time and too little leads to ambiguous results.

  • Understand how to write psuedo-code

Outline

  1. What is psuedo-code?
  2. Psuedo-code or Flow chart?
  3. Psuedo-code steps
    1. Start with a "user store" - this is a simple problem statement that describes what the program or feature is going to do
    2. Identify the inputs
    3. Identify the output
    4. Identify the steps that need to be computed
    5. Translate this into computer like language
  4. Example of psuedo-code - computing the area of a rectangle 
    1. Ask the length of the first side, store as side1 
    
				2. Ask length of the second side, store as side2 
    
				3. set "area" to be side1 * side2
				4. Return "area" to user
  5. Example - calculate a running sum. If a negative number is entered stop the computation.
  6. Exercise - Delete records that have missing values from a csv file
  7. Discussion -
    • Why do we use psuedo-code?
    • What are some other alternatives to writing psuedo code?
    • Is this somethig that people really do?
