Daily Topic - Variables

Variables are the basis of storing information while a program is running. They are the fundamental building blocks of programming. Understanding variables - name, value, and type - will essential to understanding how to write effective programs.

  • Names
    • Rule 1 - Code is read and revised more often than it is written. This means that you should optimize for readability not less typing.
    • Rule 2 - Variable names serve as the main source of information about what your code does. The ideal to strive for is that the code is "self-documenting" meaning that the code does not require any additional comments to explain its purpose or procedure. This does not mean that you should not write comments. It simply means that comments should be reserved for why you did a certain thing rather than what is going on. 
      			 
	Names Example
	da = 3
	ha = da * 7
	print(ha)
      Compare this to: 
      			
	dog_age = 3
	human_equiv_age = dog_age * 7
	print(human_equiv_age)
    • Rule 3 - Variables names should start with a letter or "_" and may contain letters, number, or underscores. (Applies to Python only)
    • Guideline 4 - Use variable names for everything in your code. (Even if they will never change). A number that is not assigned to a variable name is often called a "magic number" Strings can have a similar problem. Avoid magic variables even when the number is a fixed and known constant.

      See the example in Rule 2 - The "7" on the second line is a magic number. We should fix that by giving it a name.

      Another example is often see when computing the area of a circle. The formula is area = π * r^2. We could write this as: 

      			
	area = 3.14 * radius**2
      It would be better as: 
      			
	pi = 3.14
	area = pi * radius**2
    • Naming variables has often been called the hardest problem in computer science.
  • Type
    • A type refers tp the type of information that a variable stores.
    • Python has 4 basic variable types:
      1. Integer (int)
      2. Floatint Point Number - called a Float (float)
      3. A true/false value - called a Boolean (Bool)
      4. A sequence of characters - called a String (str)
    • Type information is stored with the variable
    • The type of a variable can not be changed. (variables names can be recycled however)
    • Variables are stored in a table like structure that holds:
      Example of Variable storage
      Name Type (length) Address in Memory
      dog_age int AC3F760D
      dog_name String (64) AC3F7612
      . . .
    • Python handles type infomration auto-magically. (One of the nice things about Python.)
      This does not mean that Python doesn't have types or is untyped. It simply means that Python enspects the variable and automatically determines the best type to store a variable as. This works 99% of the time, but sometimes it can get confused.
    • Use type(variable_name) to find the type of a variable in your code or during debugging.
    • You have to explicitly convert a variable of one type to another. This is called "casting".
      Consider the following case: The number "1" is stored as a string. You can convert it to an integer so you can do math on it by sing int("1").
      The full list of commands for casting is:
      • int ( . )
      • float ( . )
      • bool ( . )
      • str ( . )
    • Python allows you to make your own types as well has some built in higher order types. (More about this later)
  • Value
    • Can be either a bare value called a "literal" or an expression. i.e. sum = 1+2
    • Expressions depend on the type:
      • Integers and Floats - All basic mathematical operations - +, -, *, /, %, **
      • Strings - combine or concatenate 2 strings (+), repeat a string (*)
      • Boolean - boolean arithmatic ( and, or, not - covered later in class)
    • Custom operations called functions are possible - More in a future class
